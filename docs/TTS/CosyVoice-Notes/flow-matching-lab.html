Animated Intuition

Flow Matching：把每一步都看清楚

拖动时间轴观察完整链路：先从随机噪声 x0 出发，构造中间态 xt；再把 speech token、speaker prompt、mask 等条件送入速度模型 vθ；最后用 Euler / ODE 步进把 xt 推成目标 Mel x1。

生成进度：x0 -> xt -> x1 t = 35%
读公式和代码解释
1. 起点：采样噪声 x0
推理开始时还没有声学结构，只是一张随机 Mel 形状的连续变量。
2. 中间态：构造 xt
训练时从 x0 和真实 Mel x1 插值得到 xt，让模型学会任意时刻的方向。
3. 条件：内容和音色约束
speech token 决定要说什么，speaker prompt / masked Mel 约束音色和已知帧。
4. 速度：预测 vθ(xt, t, cond)
模型不是直接吐最终图，而是在当前位置预测“下一小步往哪里走”。
5. 采样：Euler / ODE 多步前进
推理时重复更新 xt = xt + vθ · Δt，逐渐形成清晰的 Mel 纹理。
6. 输出：Mel -> Vocoder -> waveform
Flow Matching 只生成 Mel；真正可播放的波形由 Vocoder 生成。
Why It Matters

放回 CosyVoice 里看

Flow Matching 不是孤立算法，它是 LLM 之后的声学生成层。LLM 先给出离散 speech token，Flow Matching 再把 token、speaker prompt、文本上下文变成可被 Vocoder 播放的 Mel。

CosyVoice 1

离线 CFM：重点是把监督语义 token 映射到高质量 Mel。

CosyVoice 2

chunk-aware CFM：限制未来信息，让 Mel 可以分块生成，降低首包延迟。

CosyVoice 3

DiT-based CFM：用更强的 estimator 承接多语种、大数据和后训练需求。
