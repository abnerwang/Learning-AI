Flow Matching：学习一条从噪声到 Mel 的路

拖动时间轴观察中间态：左侧是随机噪声 x0，中间是模型学到的速度场，右侧是目标 Mel x1。CosyVoice 的 CFM/DiT decoder 就是在文本、speaker prompt 和 speech token 条件下学习这条路。

从 noise 到 mel 的时间 t = 35%
读公式和代码解释
1. 采样噪声 x0
推理开始时还没有 Mel，只是一团随机连续变量。
2. 条件速度场 vθ
模型根据文本、prompt、speech token 判断每个点下一步往哪里走。
3. Euler / ODE 采样
多次沿速度场前进，逐步形成声学结构。
4. 得到 Mel，再交给 Vocoder
Flow Matching 产出连续声学特征，Vocoder 才负责生成最终波形。
Why It Matters

放回 CosyVoice 里看

Flow Matching 不是孤立算法，它是 LLM 之后的声学生成层。LLM 先给出离散 speech token，Flow Matching 再把 token、speaker prompt、文本上下文变成可被 Vocoder 播放的 Mel。

CosyVoice 1

离线 CFM：重点是把监督语义 token 映射到高质量 Mel。

CosyVoice 2

chunk-aware CFM：限制未来信息，让 Mel 可以分块生成，降低首包延迟。

CosyVoice 3

DiT-based CFM：用更强的 estimator 承接多语种、大数据和后训练需求。
